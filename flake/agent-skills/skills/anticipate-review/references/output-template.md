# レビュー結果の出力フォーマット

team-review が生成するレビュー結果の Markdown フォーマット。

## 完全テンプレート

```markdown
## team-review: セルフレビュー結果

**対象**: `<base-branch>...<current-branch>` / `<N>` ファイル変更 (`+<A>` / `-<B>`)
**観点カタログ**: criteria-catalog.md v<version> (<観点数> 観点)
**適用カテゴリ**: <この diff に適用した観点カテゴリのリスト>

---

## 指摘事項

### [High] C-003: 将来の変更漏れリスク

- **ファイル**: `<横断的な仕組みを実装するファイル>` (`<行番号>`)
- **問題**: 認証インターセプタが一部の RPC にのみ手動で適用されており、新規 RPC 追加時に適用漏れが起きやすい
- **理由**: 将来 RPC を追加するたびに同じ設定が必要になるパターン。デフォルト適用の仕組みに変更することで漏れを防げる
- **推奨**: デフォルトインターセプタとして全 RPC に適用される形に変更することを検討してください
- **観点 ID**: [C-003]

### [Medium] C-001: 既存パターンとの整合性

- **ファイル**: `<同種実装を持つファイル>` (`<行番号>`)
- **問題**: ページネーション方式が既存の `<同種の既存関数>` と異なっている
- **理由**: 同種実装が既に存在する場合は揃える方が望ましいパターン。一括変換時の取りこぼしリスクになる
- **観点 ID**: [C-001]

---

## 指摘なしのカテゴリ

| カテゴリ                           | 結果                 |
| ---------------------------------- | -------------------- |
| C-002: lint / 静的解析の例外設定   | 問題なし / 該当なし  |
| C-004: エラー・ログ出力の分類      | 問題なし / 該当なし  |
| C-005: コンテナイメージタグの固定性| 問題なし / 該当なし  |

---

## サマリ

| 重要度   | 件数  |
| -------- | ----- |
| High     | 1     |
| Medium   | 1     |
| **合計** | **2** |

> **次のステップ**: 上記指摘を確認後、修正 → テスト実行 → push の順で進めてください。

```

## フォーマットのルール

### 必須フィールド

各指摘には必ず以下を含める:

- **重要度**: `[High]` / `[Medium]` / `[Low]` + 観点 ID (`C-001` 等)
- **ファイル**: ファイルパスと行番号ヒント
- **問題**: 何が問題か（現象）
- **理由**: なぜ問題か（これが指摘の質を決める最重要フィールド）
- **観点 ID**: 末尾に `[観点ID]` を明記

### 推奨修正は任意

コードの修正例は、明確に書ける場合のみ記載。書けない場合は省略して OK。

### 指摘なしカテゴリの明示

レビューした結果「問題なし」と判断したカテゴリも必ず一覧にする。これにより「照合した観点が明確」になる。

### サマリは必須

最後に重要度別の件数サマリを必ず入れる。

## 指摘がゼロの場合

```markdown
## team-review: セルフレビュー結果

**対象**: `main...<branch>` / `N` ファイル変更
**適用カテゴリ**: <カテゴリ一覧>

観点照合の結果、指摘事項は見つかりませんでした。

| カテゴリ                          | 結果                |
|-----------------------------------|---------------------|
| C-001: 既存パターンとの整合性     | 問題なし / 該当なし |
| C-002: lint 例外設定              | 問題なし / 該当なし |
| C-003: 将来の変更漏れリスク       | 問題なし / 該当なし |
| C-004: エラー・ログ出力の分類     | 問題なし / 該当なし |
| C-005: コンテナイメージタグ固定性 | 問題なし / 該当なし |
```

## オペレーション系判定時

`references/scope-rules.md` の出力例を使う。指摘一覧は不要。
