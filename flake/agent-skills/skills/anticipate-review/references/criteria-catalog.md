# Review Criteria Catalog

対象 org の過去 PR レビューから抽出した観点の一覧。
このファイルを読んだ後、diff の性質に合わせて `references/criteria.md` を読む。

> **データソース**: 1000 件規模のレビューコメントから抽出（2021–2026 年）、対象レビュアー 1 人分

## 観点インデックス

| ID    | カテゴリ                     | 重要度 | 適用対象                         | 詳細              |
| ----- | ---------------------------- | ------ | -------------------------------- | ----------------- |
| C-001 | 既存パターンとの整合性       | Medium | すべての変更                     | criteria.md#c-001 |
| C-002 | lint / 静的解析の例外設定    | Medium | lint 抑制コメントを含む diff     | criteria.md#c-002 |
| C-003 | 将来の変更漏れリスク         | High   | 横断的な仕組みを導入する変更     | criteria.md#c-003 |
| C-004 | エラー・ログ出力の分類       | Low    | Go コードのエラーハンドリング    | criteria.md#c-004 |
| C-005 | コンテナイメージタグの固定性 | Medium | Docker / k8s / CI のイメージ参照 | criteria.md#c-005 |

## 重要度の定義

| 重要度 | 意味                                                                     |
| ------ | ------------------------------------------------------------------------ |
| High   | セキュリティ・認証漏れ・将来の適用漏れなど、見落とすと本番影響が出る問題 |
| Medium | 保守性・再現性・一貫性に影響するが、即時の機能障害にはならない問題       |
| Low    | スタイル・分類の問題。将来の保守コストになりうる                         |
